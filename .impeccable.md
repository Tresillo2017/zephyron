# Zephyron Design Context

## Users

**Primary audience:** The full spectrum of electronic music culture — DJs, producers, festival/club fans, and casual electronic music enthusiasts. Users come to Zephyron to stream curated DJ sets, discover new music through AI-detected tracklists, relive festival moments, and contribute to community-verified track identification.

**Usage contexts:** Late-night listening sessions, commuting, background focus music, active discovery/crate-digging, and post-event set hunting. The interface must work equally well for lean-back passive listening and engaged active browsing.

## Brand Personality

**Three words:** Futuristic. Technical. Immersive.

**Voice & tone:** Confident and knowledgeable without being pretentious. Speaks the language of electronic music culture — understands the difference between a Boiler Room set and a Tomorrowland mainstage moment. Respects both underground credibility and mainstream appeal.

**Emotional goals:**
- **Immersion** — lost in the music, the interface disappears
- **Discovery** — the thrill of finding the next track, the crate-digger's rush
- **Calm focus** — lean-back listening without distraction
- **Community belonging** — contributing to something, shared knowledge

## Aesthetic Direction

**Visual tone:** Clean with expressive touches. Dark, Spotify-adjacent foundation with moments of energy — waveform visualizations, subtle glow effects on interactive elements, smooth transitions. The technical/data-rich aspects (waveforms, tracklist timelines, confidence scores) should feel native and purposeful, not bolted on.

**References:**
- **Mixcloud** — DJ-focused, tracklist-centric layout, community feel
- **SoundCloud** — Waveform-forward, creator-focused, social discovery
- Take the best of both: Mixcloud's content focus + SoundCloud's visual energy

**Anti-references (what Zephyron is NOT):**
- Overly corporate/sterile — no enterprise software feel, must have soul
- Too playful/childish — no bubbly rounded corners, bright pastels, cartoon energy
- Too cluttered/busy — information density must be intentional, not overwhelming

**Theme:** Dark mode only. The near-black palette (`#0a0a0a` surface) evokes late-night club environments. The green accent (`#1db954`) provides energetic punctuation.

## Design Principles

### 1. Music First, Chrome Second
The audio content is the star. UI chrome (navigation, controls, metadata) should support the listening experience without competing for attention. When a set is playing, the interface should feel like it recedes.

### 2. Dense but Breathable
Electronic music culture values depth — tracklists, timestamps, artist connections, genre tags. Show this richness but with intentional spacing and hierarchy. Every piece of information earns its place.

### 3. Expressive Restraint
The dark palette provides a canvas for moments of visual energy: waveform animations, the green accent pulse on the now-playing indicator, smooth seek interactions. These expressive moments should feel earned, not gratuitous.

### 4. Progressive Disclosure
Surface the essential (set title, artist, play button) immediately. Reveal the technical (detection confidence, Last.fm links, annotation tools) on interaction. Respect both the casual listener and the deep-diver.

### 5. Community as a Feature, Not a Burden
Track corrections, votes, and annotations should feel like natural extensions of the listening experience — not separate workflows. The interface should make contributing feel effortless and rewarding.

## Component Patterns

- **Buttons:** Pill-shaped (`rounded-full`), 4 variants (primary/secondary/ghost/danger), 3 sizes
- **Cards:** `rounded-lg`, hover-reveal play buttons with translate animation, cover art + info below
- **Badges:** Pill-shaped, 3 variants (default/accent/muted), used for genres, status, confidence
- **Layout:** Persistent bottom player bar, top header with nav, content area scrolls independently
- **Grids:** Responsive 2→5 column grids for set cards
- **Typography scale:** xs (metadata) → sm (body) → base (actions) → lg/xl (section headers) → 3xl+ (page titles)

## Technical Stack

- React 19 + Vite 7 + Tailwind CSS 4 (dark theme via `@theme`)
- Zustand 5 for state (player, auth)
- Cloudflare Workers + D1 + R2
- Inter font family
- No external component library — custom primitives (Button, Badge, Input, Modal, Skeleton)

## Accessibility Requirements

- **WCAG AA** compliance for contrast ratios
- Keyboard navigation support
- `prefers-reduced-motion` respect for animations
- Semantic HTML structure
- ARIA labels for interactive elements without visible text (player controls, vote buttons)
