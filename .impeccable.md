# Zephyron Design Context

## Users

**Primary audience:** The full spectrum of electronic music culture — DJs, producers, festival/club fans, and casual electronic music enthusiasts. Users come to Zephyron to stream curated DJ sets, discover new music through AI-detected tracklists, relive festival moments, and contribute to community-verified track identification.

**Usage contexts:** Late-night listening sessions, commuting, background focus music, active discovery/crate-digging, and post-event set hunting. The interface must work equally well for lean-back passive listening and engaged active browsing.

## Brand Personality

**Three words:** Futuristic. Technical. Immersive.

**Voice & tone:** Confident and knowledgeable without being pretentious. Speaks the language of electronic music culture — understands the difference between a Boiler Room set and a Tomorrowland mainstage moment. Respects both underground credibility and mainstream appeal.

**Emotional goals:**
- **Immersion** — lost in the music, the interface disappears
- **Discovery** — the thrill of finding the next track, the crate-digger's rush
- **Calm focus** — lean-back listening without distraction
- **Community belonging** — contributing to something, shared knowledge

## Aesthetic Direction

**Visual tone:** Dark-first design inspired by the bleh Last.fm redesign. Full-bleed hero banners with real photography (not blurred), bottom-only gradients. HSL-parametric color system driven by a single `--hue` variable. Glass/blur effects on overlays and menus. Inset shadow borders on cards instead of CSS borders. No visible white borders anywhere.

**References:**
- **bleh for Last.fm** — HSL color system, card styling, banner treatment, glass menus
- **Mixcloud** — DJ-focused, tracklist-centric layout, community feel
- **SoundCloud** — Waveform-forward, creator-focused, social discovery

**Anti-references (what Zephyron is NOT):**
- Overly corporate/sterile — no enterprise software feel, must have soul
- Too playful/childish — no bubbly rounded corners, bright pastels, cartoon energy
- Too cluttered/busy — information density must be intentional, not overwhelming

## Color System (HSL-Parametric)

All colors derived from a single `--hue` CSS variable (default: 255 violet). Changing `--hue` shifts the entire UI — surfaces, text, accents, borders all shift together.

### Theme Variants (4)
| Theme | Root BG (`--b6`) | Card BG (`--b5`) | Description |
|-------|-----------------|-----------------|-------------|
| Dark (default) | 14% L | 18% L | Standard dark, balanced contrast |
| Darker | 6% L | 9% L | Deep blacks, compressed scale |
| OLED | 0% L | 4% L | True black for AMOLED screens |
| Light | 94% L | 100% L | Inverted for daytime use |

### Accent Presets (10)
Violet (255), Blue (220), Cyan (190), Teal (170), Green (145), Yellow (50), Orange (25), Red (0), Pink (330), Rose (350)

### Background Scale
- `--b6`: Root background (~14% L in dark)
- `--b5`: Card surface (~18% L)
- `--b4`: Borders, overlays (~26% L)
- `--b3`: Highlights, hover states (~30% L)
- `--b2`: Bright highlights (~40% L)

### Text Scale
- `--c1`: Primary text (near white, ~100% L, 40% S)
- `--c2`: Secondary text (~87% L, 30% S)
- `--c3`: Muted text (~60% L, 7% S)

### Accent Scale
- `--h2`: Light accent (~83% L, 50% S)
- `--h3`: Primary accent (~70% L, 60% S)
- `--h4`: Dark accent (~50% L, 70% S)

## Design Principles

### 1. Music First, Chrome Second
The audio content is the star. UI chrome (navigation, controls, metadata) should support the listening experience without competing for attention. When a set is playing, the interface should feel like it recedes.

### 2. Dense but Breathable
Electronic music culture values depth — tracklists, timestamps, artist connections, genre tags. Show this richness but with intentional spacing and hierarchy. Every piece of information earns its place.

### 3. Expressive Restraint
The dark palette provides a canvas for moments of visual energy: waveform animations, the accent pulse on the now-playing indicator, smooth seek interactions. These expressive moments should feel earned, not gratuitous.

### 4. Progressive Disclosure
Surface the essential (set title, artist, play button) immediately. Reveal the technical (detection confidence, Last.fm links, annotation tools) on interaction. Respect both the casual listener and the deep-diver.

### 5. Community as a Feature, Not a Burden
Track corrections, votes, and annotations should feel like natural extensions of the listening experience — not separate workflows. The interface should make contributing feel effortless and rewarding.

## Navigation Pattern

- **Top navigation bar** (not sidebar) — transparent on banner pages, glass backdrop on others
- **User dropdown menu** from top-right avatar (glass blur, menu shadow, solarium animation)
- **Tab bars** on detail pages (Artist, Settings) with bottom-border active indicator
- **Full-width banners** on Artist and Set pages with overlapping cover art/avatar

## Component Patterns

- **Buttons**: 12px radius (`--button-radius`), 30px height (`--button-height`), `scale(0.98)` on active, accent glow shadow on primary. 4 variants: primary/secondary/ghost/danger. 3 sizes: sm/md/lg.
- **Cards**: `.card` utility class — `hsl(var(--b5))` bg, `inset 0 0 0 1px hsl(var(--b4) / 0.25)` border (inset shadow, NOT CSS border), 12px radius, 20px padding, `0 8px 35px rgba(0,0,0,0.35)` shadow.
- **Tags**: `#` prefixed (e.g. `#house`), low-opacity bg `hsl(var(--b4) / 0.4)`, 6px radius, no border.
- **Badges**: accent/default/muted variants, no inset shadow borders.
- **Toggles**: 42x24px switch with white 18px knob, accent bg when checked, `hsl(var(--b3))` when unchecked.
- **Sliders**: Accent-filled track with `hsl(var(--b3))` unfilled, white 16px round thumb with subtle shadow.

## Layout Patterns

- **Top nav**: `h-[calc(20px + var(--button-height))]`, transparent bg, glass backdrop on non-banner pages, sticky `top-0 z-40`
- **Player bar**: Glass backdrop (`hsl(var(--b5) / 0.9)`), 72px height, accent play button (9x9, rounded-full) with glow shadow, centered controls with progress bar, left (set info 240px) / center (controls flex-1 max-w-640px) / right (volume + expand 240px)
- **Banner pages**: 280px full-bleed hero, bottom-only gradient (`from-surface via-surface/30 to-transparent`), -100px content overlap, cover art 130-160px with `var(--subtle-shadow)`
- **Content pages**: `px-6 lg:px-10 py-6` padding, no max-width constraint
- **Settings**: Vertical tab nav on left (200px), content on right, max-w-2xl
- **Two-column detail pages**: Main content (flex-1) + sidebar (300-340px) with `.card` sections

## Typography

- **Font**: Geist (variable weight 100-900), Geist Mono for timestamps/data
- **Weight scale**: 480 (body, `--font-weight`), 650 (medium, `--font-weight-medium`), 730 (bold, `--font-weight-bold`)
- **Scale**: xs (metadata/timestamps) → sm (body) → base (actions) → lg (section headers) → 2xl-4xl (page titles)
- **Mono**: Font-mono for timestamps, play counts, confidence scores, hue values

## Motion

- **Transitions**: 0.2s with `cubic-bezier(0.095, 0.41, 0.055, 0.96)` (`--ease-out-custom`)
- **Spring easing**: `cubic-bezier(0.205, 0.06, 0.05, 0.965)` for entrances
- **Equalizer bars**: 3-bar animation at different rates (0.6-0.8s) for now-playing indicators
- **Solarium**: Scale + blur reveal animation for menus/modals
- **Reduced motion**: All animations disabled via `prefers-reduced-motion`

## Technical Stack

- React 19 + Vite 7 + Tailwind CSS 4 (`@theme` + CSS custom properties)
- Zustand 5 for state (player, auth, theme)
- Cloudflare Workers + D1 + R2 + Workers AI + Queues + Durable Objects + Vectorize
- Geist font family (variable weight)
- Better Auth with admin plugin
- No external component library — custom primitives (Button, Badge, Input, Modal, Skeleton, Toggle, Slider, TabBar)

## Accessibility Requirements

- **WCAG AA** compliance for contrast ratios
- Keyboard navigation support
- `prefers-reduced-motion` respect for all animations
- Semantic HTML structure
- ARIA labels for interactive elements without visible text (player controls, vote buttons)
- Banner text uses `text-shadow` for readability over photos
